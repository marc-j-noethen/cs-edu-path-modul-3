# CS Edu Path Module 3

Module 3 focuses on cloud and enterprise security, as well as the transition to DevSecOps and Infrastructure as Code. This repository brings together the learning content for Azure fundamentals, governance, monitoring, containers, CI/CD, software supply chain security and Terraform.

## Overview

| Area | Content |
|---|---|
| Module objective | Build an understanding of security for modern cloud and enterprise environments |
| Focus | Azure, Cloud Governance, DevSecOps, Containers, IaC |
| Structure | Weekly folders with pre-class material, summaries and exercises |
| Scope | 5 weekly folders, 18 active topic folders with PC documents and summaries |

## Folder Structure

Most topics in Module 3 follow a three-part structure:

```text
cs-edu-path-modul-3/
├── 01-Week-.../
│   ├── 01-Intro-to-Cloud/
│   │   ├── 0_PC-Document/        (optional, not present in every topic)
│   │   ├── 1-Concepts-Tools-Terms/
│   │   └── 2-Excersises/         (legacy folder name kept as-is in this repository)
│   └── ...
├── 02-Week-.../
├── ...
└── README.md
```

## Meaning of the standard folders

- `0_PC-Document/`
  Optional pre-class or input materials from which the thematic summaries were derived.
- `1-Concepts-Tools-Terms/`
  Structured learning summaries, technical terms, tools and vocabulary for the respective topic.
- `2-Excersises/`
  Exercise and assignment materials for practical consolidation. The spelling is inconsistent in the repository, but the folder name is preserved to avoid breaking links.

In addition, individual weekly folders may contain help or supplementary directories, for example `week1-guides`, `week2-github-guides` or career/support materials.

## What this module covers

By the end of Module 3, you should:

- understand key Azure concepts for compute, networking, IAM, storage and governance
- be able to contextualise cost, monitoring and security aspects in cloud environments
- be able to evaluate CI/CD and DevSecOps pipelines from a security and operational perspective
- be able to classify containers, registries and supply chain risks accurately
- understand the basics of Infrastructure as Code with Terraform

## How to use

- Start with  `1-Concepts-Tools-Terms` as a condensed basis for learning and revision.
- Then work through `2-Exercises` to consolidate your understanding of the concepts in a practical way.
- The summaries in the Concepts folders are particularly useful for quick revision.

## Special features of this repository

- The `Concepts-Tools-Terms.md` files have already been organised into a structured 80/20 learning format for several topics.
- Module 3 is the most cloud- and process-oriented section of the path to date and links technical platform topics directly to Security Operations and DevSecOps.

## Position within the overall path

Module 3 builds on the technical foundations from Module 1 and the security practices from Module 2. It combines both in modern enterprise and cloud contexts, thereby serving as a bridge towards specialised security and cloud roles.
